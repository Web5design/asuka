asuka.instance — Instances¶

class asuka.instance.Instance(app, instance, login)¶

Thin abstraction layer upon boto.ec2.instance.Instance object. It’s a combination of those three fields:

  • app
  • instance
  • login

It also provides SSH/SFTP connection to the instance using paramiko module under the hood. There are convenient methods like do(), put_file(), remove_file() and context managers to control sessions.

For example, the following code connects to the instance twice:

instance.do('echo 1')
instance.do('echo 2')

If you pass the instance into with statement, it keeps the connection session until the block ends. For example, the following code connects to the instance only once:

with instance:
    instance.do('echo 1')
    instance.do('echo 2')

Or if you use it with as, you can deal with the law-level paramiko.client.SSHClient object:

with instance as ssh:
    in_, out, err = ssh.exec_command('echo 1')
    print out.readline()
    print >> sys.stderr, err.readline()
    in_.close()
    out.close()
    err.close()

It automatically check the status of the instance to make sure it’s available to connect, and waits until it’s running. You can explicitly wait the instance using wait_state() method:

instance.wait_state()
with instane:
    instance.do('echo 1')

— but you don’t have to of course.

app = None¶

(App) The app object.

do(command, environ={})¶

Executes the given command on the SSH connection session. If there’s no currently running session, it implictly connects to the instance.

The command can be a raw string or a sequence of commands to be quoted:

instance.do('echo "Hello world"')
instance.do(['echo', 'Hello world'])

It can take environment variables to set:

instance.do('date', environ={'LANG': 'ko_KR'})
Parameters:
  • command (basestring, collections.Sequence) – the command to execute. if it isn’t string but sequence, it becomes quoted and joined
  • environ (collections.Mapping) – optional environment variables
instance = None¶

make_directory(path, mode=493, sudo=False)¶

Creates the path directory into the remote.

Parameters:
  • path (basestring) – the remote path of the directory to create
  • mode (numbers.Integral) – the permission mode of the directory e.g. 0755
  • sudo (bool) – as superuser or not. default is False
open_file(*args, **kwds)¶

Opens the remote file as context manager:

with instance.open_file('/tmp/test', 'w') as:
    print >> f, 'hello world'
with instance.open_file('/tmp/test') as:
    print f.read()
Parameters:
  • path (basestring) – the remote path of the file to open
  • mode (basestring) – the opening mode e.g. 'r', 'wb'
put_directory(local_path, remote_path, sudo=False)¶

Uploads the local_path directory to the remote_path.

Parameters:
  • local_path (basestring) – the local path to upload
  • remote_path (basestring) – the remote path
  • sudo (bool) – as superuser or not. default is False
put_file(local_path, remote_path, sudo=False)¶

Uploads the local_path file to the remote_path.

Parameters:
  • local_path (basestring) – the local path to upload
  • remote_path (basestring) – the remote path
  • sudo (bool) – as superuser or not. default is False
remove_file(path, sudo=False)¶

Deletes the path from the remote.

Parameters:
  • path (basestring) – the path to delete
  • sudo (bool) – as superuser or not. default is False
sftp(*args, **kwds)¶

Opens the context manager of SFTP session. For example:

with instance.sftp():
    instance.put_file('local_file', 'remote_file')

It yields the low-level paramiko.sftp_client.SFTPClient as well:

with instance.sftp() as sftp:
    print sftp.getcwd()
sudo(command, environ={})¶

The same as do() except the command is executed by superuser.

wait_state(state='running', timeout=60, tick=5)¶

Waits until the instance state becomes to the given goal state (default is 'running').

Parameters:
  • state (basestring) – the goal state to wait. default is 'running'
  • timeout (numbers.Real) – the timeout in seconds. default is 60 seconds
  • tick (numbers.Real) – the tick seconds to refresh. default is 5 seconds
write_file(path, content, sudo=False)¶

Writes the content to the remote path. Useful for saving configuration files.

Parameters:
  • path (basestring) – the remote path to write
  • content (str) – the file content
  • sudo (bool) – as superuser or not. default is False
